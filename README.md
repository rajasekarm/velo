**Rajasekar's high-velocity engineering team, running on Claude Code.**

Velo is Rajasekar's agentic engineering team — a full squad of specialised Claude agents coordinated by an Engineering Manager. Describe what you want built. Velo plans it, assigns the right engineers, runs work in parallel, and ships with review baked in.

No bottlenecks. No context-switching. Just output.

## How it works

- You give Velo a task — a feature, a fix, a refactor, anything
- Velo assesses scope and decides whether to plan first or build immediately
- A Product Manager clarifies requirements; a Spec Writer locks in the technical design
- Builder agents (FE, BE, DB, Infra, Automation) execute their scoped work in parallel where possible
- Dependent work is sequenced automatically — DB before BE, BE before FE
- Reviewers for each domain run after build, catching issues before they ship
- Velo synthesises everything into a single summary report

| **Reviewer** | Backend Reviewer | API design, error handling, security |
| **Reviewer** | Database Reviewer | Schema correctness, index coverage, query safety |
| **Reviewer** | Infrastructure Reviewer | Config hygiene, security posture, cost |
| **Reviewer** | Automation Reviewer | Test coverage, reliability, flakiness |
| **Utility** | Commit Agent | Analyse diff, generate commit message, create git commit |

## Why Velo?

- **High velocity**: Parallel execution across all independent domains — frontend, backend, and infra ship simultaneously, not sequentially.
- **No skipped steps**: Planning before building. Review before done. Process is enforced, not optional.
- **Right engineer for the job**: Each agent has a scoped role and domain-specific knowledge. Nobody wanders outside their lane.
- **You set direction, Velo handles coordination**: One task in, a shipped feature out.
