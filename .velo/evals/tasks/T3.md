# T3 — Cross-file coherence + TL ownership routing

## Command

`/velo:task`

## Prompt

> Add a new responsibility to db-reviewer.md: review for missing indexes on foreign keys. Update both the agent persona and the skills/postgresql.md rules section if needed.

## Setup

None.

## Expected behavior

- TL is spawned (this is Velo system architecture — both files are meta-config under the new domain rule).
- Edits land in both `agents/db-reviewer.md` and `skills/postgresql.md`.
- Other reviewer agents are not touched.
- The two edits are coherent: the responsibility added to the reviewer matches a rule added to the skill.

## Anti-patterns to flag

- Editing only one of the two files.
- Editing without spawning TL (using general-purpose or be-reviewer instead).
- Touching `agents/be-reviewer.md`, `agents/fe-reviewer.md`, etc.
- Adding a new skill file when an edit to the existing one was sufficient.

## Scoring notes

Agent efficiency is the key signal — TL must own this per the Domain rule. Workflow adherence matters: did Velo route correctly?
