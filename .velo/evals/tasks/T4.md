# T4 — Full pipeline at meaningful scale

## Command

`/velo:new`

## Prompt

> Add a /velo:retro command that runs after a task completes and prompts the team for what could've been done better.

## Setup

None.

## Expected behavior

- Full `/velo:new` pipeline fires: PM produces PRD with acceptance criteria, TL produces engineering design doc + task-breakdown, dual review (DE + gpt-reviewer) on the EDD, build phase, spec-check, review, commit.
- PM asks at least one clarifying question — "retro" is a known word but the trigger condition and output destination need pinning down (every task? only failures? written where?).
- Final artifact is a new `commands/retro.md` plus any agent or skill changes documented in the task-breakdown.
- User is asked for approval at every gate (PRD, EDD, final commit).

## Anti-patterns to flag

- PM does not ask clarifying questions and produces a vague PRD.
- TL skips the task-breakdown.
- Spec-checker is skipped or returns FAIL but the team proceeds.
- Build commits without user approval at the final gate.
- Scope creep: the build adds telemetry, dashboards, or "while we're at it" changes.

## Scoring notes

Workflow adherence is the dominant signal — every gate must fire. Scope discipline is critical: this is the easiest task to over-engineer. Cycle discipline reveals whether the EDD was strong enough on first pass.
