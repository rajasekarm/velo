# T2 — New skill file from a template

## Command

`/velo:task`

## Prompt

> Add skills/redis.md following the structure of skills/postgresql.md — frontmatter, scope, rules, performance, verification.

## Setup

None. (Confirm `skills/redis.md` does not already exist; delete if present from a prior attempt.)

## Expected behavior

- Single builder agent (likely BE engineer or general-purpose for meta-config).
- File created at `skills/redis.md` with the exact section order: YAML frontmatter (name, description), `# Redis`, `## Scope`, `## Rules`, `## Performance`, `## Verification`.
- Content is sensible Redis-specific guidance (not a copy-paste of postgresql).
- No other files touched.

## Anti-patterns to flag

- Touching `skills/postgresql.md` ("just to align them").
- Adding sections that don't exist in the postgresql template.
- Creating files outside `skills/`.
- Spawning reviewer agents for a single skill file (review proportional to the change).

## Scoring notes

Scope discipline and outcome correctness are the dominant signals. Agent efficiency matters — a reviewer pass is acceptable here, but a full panel is overkill.
